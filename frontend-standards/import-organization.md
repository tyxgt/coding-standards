# 前端导入规范

## 导入分组顺序

所有 import 按以下 7 组顺序排列，组间用空行分隔，同类按字母序：

1. **核心库**：React、React Router 等
2. **第三方 UI 库**：Ant Design、ProComponents 等
3. **第三方工具库**：ahooks、classnames、date-fns 等
4. **内部别名**：`@/` 开头的导入
5. **相对路径**：同级 `./` 或父级 `../` 导入
6. **样式**：`import styles from '...'`
7. **类型导入**：`import type { ... }`

## 路径别名

| 别名 | 路径 | 说明 |
|------|------|------|
| `@/` | `src/` | 源码根目录，跨目录引用时使用 |
| `@@/` | `src/.umi/` | Umi 运行时生成目录 |

### 使用规则

- `@/` 别名用于跨目录引用（不同页面、跨大目录引用 utils/services/models）
- 同一页面目录内使用相对路径
- 相对路径不超过 3 层（`../../../`），超过则改用 `@/`
- 避免在同一个模块内混用 `@/` 和相对路径指向同一目录

### 引用方式速查

| 引用位置 | 目标位置 | 推荐方式 |
|----------|----------|----------|
| 同一目录 | 同级文件 | 相对路径 `./xxx` |
| 同一页面 | 子目录 | 相对路径 `./components/xxx` |
| 同一页面 | 父目录 | 相对路径 `../xxx` |
| 不同页面 | src 下任意位置 | `@/pages/xxx` 或 `@/components/xxx` |
| 跨大目录 | utils/services/models | `@/utils/xxx` |
| 第三方库 | - | 直接包名 |

## 路径别名配置

路径别名在 `tsconfig.json` 的 `paths` 中配置：

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

## 禁止事项

- ❌ 混用 `@/` 和相对路径指向同一目录
- ❌ 相对路径回退超过 3 层（`../../../../`）
- ❌ 循环依赖（A → B → A）
